# Bootstrap progress log and /progress skill

## What changed
- Added `.claude/skills/progress/SKILL.md` — project-level `/progress` skill that logs session entries and keeps the top-level README in sync with `origin/main`.

## Why
- Need persistent, human-readable session handoffs that survive across Claude Code sessions (memory alone is opaque and session-scoped).
- Wanted a single command (`/progress`) to (a) write session notes and (b) refresh the top-level README whenever commits land on `main`.
- CLAUDE.md is loaded every session, so that's the cheapest way to guarantee the folder is read at startup.
